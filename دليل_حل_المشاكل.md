# 🔧 دليل حل المشاكل الشائعة

## 🚨 المشاكل الشائعة وحلولها

### 1. خطأ 429 - Too Many Requests

**الأعراض:**
```
GET http://localhost:3000/api/... 429 (Too Many Requests)
```

**السبب:** تجاوز حد الطلبات المسموح به

**الحل:**
✅ **تم الحل!** تم زيادة الحد إلى 1000 طلب/15 دقيقة

إذا استمرت المشكلة:
1. أعد تشغيل Backend
2. امسح cache المتصفح
3. انتظر 15 دقيقة

---

### 2. فشل تسجيل الدخول

**الأعراض:**
- "بيانات الدخول غير صحيحة"
- لا يمكن الدخول

**الحلول:**

#### الحل 1: استخدم البيانات الصحيحة
```
البريد: shahidmusaa2030@gmail.com
كلمة المرور: Admin@123
```

#### الحل 2: إعادة تعيين كلمة المرور
```bash
cd backend
node -e "const bcrypt = require('bcryptjs'); const initSqlJs = require('sql.js'); const fs = require('fs'); const newPassword = 'Admin@123'; bcrypt.hash(newPassword, 12).then(hash => { initSqlJs().then(SQL => { const buffer = fs.readFileSync('./data/sponsorship.db'); const db = new SQL.Database(buffer); db.run('UPDATE admins SET password = ? WHERE email = ?', [hash, 'shahidmusaa2030@gmail.com']); const data = db.export(); fs.writeFileSync('./data/sponsorship.db', Buffer.from(data)); console.log('تم تحديث كلمة المرور!'); }); });"
```

### 3. Backend لا يعمل

**الأعراض:**
- `ECONNREFUSED`
- لا يمكن الوصول للـ API

**الحلول:**

#### تحقق من تشغيل Backend
```bash
cd backend
npm start
```

#### تحقق من المنفذ
```bash
# Windows
netstat -ano | findstr :5000

# إذا كان مشغولاً، أوقفه
taskkill /PID <رقم_العملية> /F
```

#### تحقق من الأخطاء
```bash
# افحص logs
cat backend/logs/error.log
```

---

### 4. Frontend لا يعمل

**الأعراض:**
- صفحة بيضاء
- أخطاء في Console

**الحلول:**

#### أعد تشغيل Frontend
```bash
cd frontend
npm run dev
```

#### امسح node_modules وأعد التثبيت
```bash
cd frontend
rm -rf node_modules
npm install
npm run dev
```

---

### 5. قاعدة البيانات تالفة

**الأعراض:**
- أخطاء في قراءة/كتابة البيانات
- "Database error"

**الحلول:**

#### استعادة من نسخة احتياطية
```bash
cd backend/backups
# انسخ أحدث نسخة احتياطية
copy backup_*.db ../data/sponsorship.db
```

#### إنشاء قاعدة بيانات جديدة
```bash
cd backend
# احذف القاعدة القديمة
rm data/sponsorship.db
# أعد تشغيل Backend لإنشاء قاعدة جديدة
npm start
```

---

### 6. 2FA لا يعمل

**الأعراض:**
- لا يصل رمز التحقق
- خطأ في إرسال البريد

**الحلول:**

#### تحقق من إعدادات البريد
```bash
# تحقق من .env
cat backend/.env | grep EMAIL
```

#### تعطيل 2FA مؤقتاً
1. اذهب إلى الإعدادات
2. تبويب "النظام"
3. أوقف "التحقق بخطوتين"

#### إعداد Gmail App Password
1. اذهب إلى https://myaccount.google.com/apppasswords
2. أنشئ App Password جديد
3. ضعه في `EMAIL_PASS` في .env

### 7. الملفات لا ترفع

**الأعراض:**
- خطأ عند رفع المستندات
- "Upload failed"

**الحلول:**

#### تحقق من مجلد uploads
```bash
# تأكد من وجود المجلد
mkdir backend/uploads
```

#### تحقق من الصلاحيات
```bash
# Windows
icacls backend/uploads /grant Users:F
```

#### تحقق من حجم الملف
- الحد الأقصى: 10 MB
- الأنواع المسموحة: PDF, JPG, PNG, DOCX

---

### 8. بطء في الأداء

**الأعراض:**
- التطبيق بطيء
- استجابة متأخرة

**الحلول:**

#### امسح سجل النشاطات القديم
```bash
cd backend
node -e "const initSqlJs = require('sql.js'); const fs = require('fs'); initSqlJs().then(SQL => { const buffer = fs.readFileSync('./data/sponsorship.db'); const db = new SQL.Database(buffer); db.run('DELETE FROM activity_logs WHERE created_at < datetime(\"now\", \"-30 days\")'); const data = db.export(); fs.writeFileSync('./data/sponsorship.db', Buffer.from(data)); console.log('تم مسح السجلات القديمة'); });"
```

#### امسح cache المتصفح
- Chrome: Ctrl + Shift + Delete
- اختر "Cached images and files"

---

### 9. CORS Error

**الأعراض:**
```
Access to XMLHttpRequest blocked by CORS policy
```

**الحلول:**

#### تحقق من CORS_ORIGIN
```bash
# في backend/.env
CORS_ORIGIN=http://localhost:3000
```

#### أعد تشغيل Backend
```bash
cd backend
npm start
```

---

### 10. Port Already in Use

**الأعراض:**
```
Error: listen EADDRINUSE: address already in use :::5000
```

**الحلول:**

#### Windows
```bash
# ابحث عن العملية
netstat -ano | findstr :5000

# أوقف العملية
taskkill /PID <رقم_العملية> /F
```

#### غيّر المنفذ
```bash
# في backend/.env
PORT=5001
```
