## 🎯 التحضير للنشر

### 1. فحص المشروع

قبل النشر، تأكد من:

```bash
# اختبار Backend
cd backend
npm install
npm start

# اختبار Frontend
cd frontend
npm install
npm run build
```

### 2. توليد المفاتيح الآمنة

قم بتوليد مفاتيح آمنة للإنتاج:

```bash
# JWT Secret (64 حرف)
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

# Encryption Key (32 حرف)
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

احفظ هذه المفاتيح في مكان آمن!

### 3. قائمة التحقق

- [ ] تغيير كلمة المرور الافتراضية للمدير
- [ ] توليد JWT_SECRET جديد
- [ ] توليد ENCRYPTION_KEY جديد
- [ ] إعداد بريد إلكتروني للـ 2FA
- [ ] الحصول على شهادة SSL
- [ ] تحديد CORS_ORIGIN للدومين الخاص بك

## 🖥️ النشر على VPS

### الخطوة 1: اختيار مزود VPS

مزودون موصى بهم:
- **DigitalOcean** - $6/شهر (1GB RAM)
- **Linode** - $5/شهر (1GB RAM)
- **Vultr** - $6/شهر (1GB RAM)
- **AWS Lightsail** - $5/شهر (1GB RAM)

### الخطوة 2: إعداد السيرفر

```bash
# 1. الاتصال بالسيرفر
ssh root@your_server_ip

# 2. تحديث النظام
apt update && apt upgrade -y

# 3. تثبيت Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install -y nodejs

# 4. تثبيت Nginx
apt install -y nginx

# 5. تثبيت Certbot (SSL)
apt install -y certbot python3-certbot-nginx

# 6. تثبيت PM2
npm install -g pm2
```

### الخطوة 3: رفع المشروع

```bash
# على جهازك المحلي
# 1. ضغط المشروع
tar -czf project.tar.gz backend frontend

# 2. رفع للسيرفر
scp project.tar.gz root@your_server_ip:/var/www/

# على السيرفر
cd /var/www
tar -xzf project.tar.gz
```

### الخطوة 4: إعداد Backend

```bash
cd /var/www/backend

# 1. تثبيت الحزم
npm install --production

# 2. إنشاء ملف .env
nano .env
```

انسخ المحتوى التالي وعدّل القيم:

```env
PORT=5000
NODE_ENV=production
JWT_SECRET=YOUR_GENERATED_64_CHAR_KEY
JWT_EXPIRES_IN=7d
ENCRYPTION_KEY=YOUR_GENERATED_32_CHAR_KEY
EMAIL_SERVICE=gmail
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=your_app_password
CORS_ORIGIN=https://yourdomain.com
```

```bash
# 3. تشغيل Backend مع PM2
pm2 start src/server.js --name sponsorship-backend
pm2 save
pm2 startup
```

### الخطوة 5: إعداد Frontend

```bash
cd /var/www/frontend

# 1. تثبيت الحزم
npm install

# 2. تحديث API URL
nano src/services/api.ts
# غيّر baseURL إلى: https://yourdomain.com/api

# 3. بناء المشروع
npm run build

# 4. نسخ الملفات المبنية
cp -r dist/* /var/www/html/
```

### الخطوة 6: إعداد Nginx

```bash
nano /etc/nginx/sites-available/sponsorship
```

انسخ التكوين التالي:

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    # Frontend
    location / {
        root /var/www/html;
        try_files $uri $uri/ /index.html;
    }

    # Backend API
    location /api {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # حد أقصى لحجم الملفات المرفوعة
    client_max_body_size 10M;
}
```

```bash
# تفعيل الموقع
ln -s /etc/nginx/sites-available/sponsorship /etc/nginx/sites-enabled/
nginx -t
systemctl restart nginx
```

### الخطوة 7: الحصول على SSL

```bash
# احصل على شهادة SSL مجانية من Let's Encrypt
certbot --nginx -d yourdomain.com -d www.yourdomain.com

# التجديد التلقائي
certbot renew --dry-run
```

### الخطوة 8: إعداد Firewall

```bash
ufw allow 22
ufw allow 80
ufw allow 443
ufw enable
```

## ☁️ النشر على Vercel + Railway

### الخطوة 1: نشر Backend على Railway

1. اذهب إلى [railway.app](https://railway.app)
2. سجل دخول بحساب GitHub
3. اضغط "New Project" → "Deploy from GitHub repo"
4. اختر المشروع
5. أضف المتغيرات البيئية:
   ```
   NODE_ENV=production
   PORT=5000
   JWT_SECRET=your_key
   ENCRYPTION_KEY=your_key
   EMAIL_SERVICE=gmail
   EMAIL_USER=your_email
   EMAIL_PASS=your_password
   CORS_ORIGIN=https://your-app.vercel.app
   ```
6. احفظ الـ URL الذي يعطيك إياه Railway

### الخطوة 2: نشر Frontend على Vercel

1. اذهب إلى [vercel.com](https://vercel.com)
2. سجل دخول بحساب GitHub
3. اضغط "Add New" → "Project"
4. اختر المشروع
5. في إعدادات Build:
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
6. أضف متغير بيئي:
   ```
   VITE_API_URL=https://your-railway-url.railway.app
   ```
7. اضغط "Deploy"

### الخطوة 3: تحديث API URL

عدّل ملف `frontend/src/services/api.ts`:

```typescript
const API_URL = import.meta.env.VITE_API_URL || 'http://localhost:5000';
```

## 🎨 النشر على Heroku

### الخطوة 1: إعداد Heroku

```bash
# تثبيت Heroku CLI
# Windows: قم بتحميله من heroku.com
# Mac: brew install heroku/brew/heroku
# Linux: curl https://cli-assets.heroku.com/install.sh | sh

# تسجيل الدخول
heroku login
```

### الخطوة 2: نشر Backend

```bash
cd backend

# إنشاء تطبيق Heroku
heroku create your-app-name-backend

# إضافة المتغيرات البيئية
heroku config:set NODE_ENV=production
heroku config:set JWT_SECRET=your_key
heroku config:set ENCRYPTION_KEY=your_key
heroku config:set EMAIL_SERVICE=gmail
heroku config:set EMAIL_USER=your_email
heroku config:set EMAIL_PASS=your_password

# إنشاء Procfile
echo "web: node src/server.js" > Procfile

# نشر
git init
git add .
git commit -m "Deploy backend"
git push heroku main
```

### الخطوة 3: نشر Frontend

```bash
cd frontend

# تحديث API URL في .env
echo "VITE_API_URL=https://your-app-name-backend.herokuapp.com" > .env.production

# إنشاء تطبيق Heroku
heroku create your-app-name-frontend

# إضافة buildpack
heroku buildpacks:add heroku/nodejs

# نشر
git init
git add .
git commit -m "Deploy frontend"
git push heroku main
```

## ⚙️ إعدادات ما بعد النشر

### 1. تغيير كلمة المرور الافتراضية

```bash
# سجل دخول بالبيانات الافتراضية
# ثم اذهب إلى الإعدادات → إدارة الحسابات
# وغيّر كلمة المرور
```

### 2. إعداد النسخ الاحتياطي التلقائي

النظام يقوم بنسخ احتياطي تلقائي يومياً، لكن يُنصح بإعداد نسخ احتياطي خارجي:

```bash
# على VPS - إضافة cron job
crontab -e

# أضف السطر التالي (نسخ احتياطي يومي الساعة 3 صباحاً)
0 3 * * * cp /var/www/backend/data/sponsorship.db /backup/sponsorship-$(date +\%Y\%m\%d).db
```

### 3. مراقبة الأداء

```bash
# على VPS مع PM2
pm2 monit

# عرض السجلات
pm2 logs sponsorship-backend

# إعادة التشغيل
pm2 restart sponsorship-backend
```

### 4. تحديث المشروع

```bash
# على VPS
cd /var/www/backend
git pull origin main
npm install
pm2 restart sponsorship-backend

cd /var/www/frontend
git pull origin main
npm install
npm run build
cp -r dist/* /var/www/html/
```

## 🔍 استكشاف الأخطاء

### المشكلة: Backend لا يعمل

```bash
# فحص السجلات
pm2 logs sponsorship-backend

# فحص حالة PM2
pm2 status

# إعادة التشغيل
pm2 restart sponsorship-backend
```

### المشكلة: Frontend لا يتصل بـ Backend

1. تأكد من CORS_ORIGIN في Backend
2. تأكد من API URL في Frontend
3. تأكد من SSL يعمل بشكل صحيح

### المشكلة: SSL لا يعمل

```bash
# فحص شهادة SSL
certbot certificates

# تجديد الشهادة
certbot renew

# إعادة تشغيل Nginx
systemctl restart nginx
```
