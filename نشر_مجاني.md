## ⚡ الطريقة السريعة (5 دقائق)

### الخطوة 1: تحضير المشروع

```bash
chmod +x deploy-free.sh
./deploy-free.sh
```

هذا السكريبت سيقوم بـ:
- ✅ فحص المتطلبات
- ✅ توليد مفاتيح آمنة
- ✅ اختبار البناء
- ✅ تحضير Git

### الخطوة 2: رفع على GitHub

1. اذهب إلى [github.com/new](https://github.com/new)
2. أنشئ repository جديد (اسمه مثلاً: `sponsorship-system`)
3. **لا تضف** README أو .gitignore أو license
4. انسخ الأوامر ونفذها:

```bash
git remote add origin https://github.com/username/sponsorship-system.git
git branch -M main
git push -u origin main
```

### الخطوة 3: النشر على Render

#### أ. إنشاء حساب

1. اذهب إلى [render.com](https://render.com)
2. اضغط **Get Started for Free**
3. سجل دخول بحساب GitHub
4. اسمح لـ Render بالوصول للـ repository

#### ب. نشر Backend

1. من لوحة التحكم، اضغط **New +**
2. اختر **Web Service**
3. اختر المشروع من القائمة
4. املأ البيانات:

```
Name: sponsorship-backend
Region: Frankfurt
Branch: main
Root Directory: backend
Runtime: Node
Build Command: npm install
Start Command: node src/server.js
Instance Type: Free
```

5. اضغط **Advanced** وأضف المتغيرات البيئية:

```
NODE_ENV = production
PORT = 10000
JWT_SECRET = [انسخ من ملف .deployment-keys.txt]
ENCRYPTION_KEY = [انسخ من ملف .deployment-keys.txt]
EMAIL_SERVICE = gmail
EMAIL_USER = your_email@gmail.com
EMAIL_PASS = [App Password من Gmail]
CORS_ORIGIN = https://sponsorship-frontend.onrender.com
```

6. اضغط **Create Web Service**
7. انتظر 5-10 دقائق حتى ينتهي النشر
8. **انسخ الـ URL** (مثلاً: `https://sponsorship-backend-abc123.onrender.com`)

#### ج. نشر Frontend

1. من لوحة التحكم، اضغط **New +**
2. اختر **Static Site**
3. اختر نفس المشروع
4. املأ البيانات:

```
Name: sponsorship-frontend
Region: Frankfurt
Branch: main
Root Directory: frontend
Build Command: npm install && npm run build
Publish Directory: dist
```

5. أضف متغير بيئي واحد:

```
VITE_API_URL = [الصق URL الـ Backend من الخطوة السابقة]
```

6. اضغط **Create Static Site**
7. انتظر 3-5 دقائق

#### د. تحديث CORS

1. ارجع لـ **Backend** في Render
2. اذهب إلى **Environment**
3. عدّل `CORS_ORIGIN` إلى URL الـ Frontend (مثلاً: `https://sponsorship-frontend.onrender.com`)
4. احفظ (سيعيد النشر تلقائياً)

### الخطوة 4: الوصول للنظام

افتح المتصفح على URL الـ Frontend:
```
https://sponsorship-frontend.onrender.com
```

**بيانات الدخول:**
- البريد: `admin@example.com`
- كلمة المرور: `Admin@123`

⚠️ **مهم جداً**: غيّر كلمة المرور فوراً!

## 📧 إعداد Gmail للبريد الإلكتروني

### الخطوة 1: تفعيل المصادقة الثنائية

1. اذهب إلى [myaccount.google.com](https://myaccount.google.com)
2. اضغط **الأمان** (Security)
3. اضغط **التحقق بخطوتين** (2-Step Verification)
4. اتبع الخطوات لتفعيله

### الخطوة 2: إنشاء App Password

1. ارجع لـ **الأمان**
2. اضغط **كلمات مرور التطبيقات** (App passwords)
3. اختر **البريد** (Mail)
4. اختر **أخرى** (Other) واكتب: "Sponsorship System"
5. اضغط **إنشاء** (Generate)
6. **انسخ** كلمة المرور (16 حرف)
7. استخدمها في `EMAIL_PASS` على Render

## 🔄 منع التوقف التلقائي

الخطة المجانية تتوقف بعد 15 دقيقة من عدم النشاط. لمنع ذلك:

### استخدام UptimeRobot (مجاني)

1. اذهب إلى [uptimerobot.com](https://uptimerobot.com)
2. سجل حساب مجاني
3. اضغط **Add New Monitor**
4. املأ البيانات:

```
Monitor Type: HTTP(s)
Friendly Name: Sponsorship Backend
URL: https://your-backend.onrender.com/api/health
Monitoring Interval: 5 minutes
```

5. احفظ

الآن الموقع سيبقى نشطاً دائماً! 🎉

## 📱 ربط نطاق مخصص (اختياري)

### نطاق مجاني من Freenom

1. اذهب إلى [freenom.com](https://www.freenom.com)
2. ابحث عن نطاق متاح (مثلاً: `myapp.tk`)
3. سجل مجاناً
4. في Render، اذهب إلى **Settings** → **Custom Domain**
5. أضف النطاق
6. في Freenom، أضف CNAME record:
   ```
   Type: CNAME
   Name: www
   Target: your-app.onrender.com
   ```

## 🆘 حل المشاكل

### المشكلة: Backend بطيء في أول زيارة

**السبب**: الخطة المجانية تتوقف بعد 15 دقيقة

**الحل**: استخدم UptimeRobot كما شرحنا أعلاه

### المشكلة: CORS Error

**السبب**: `CORS_ORIGIN` غير صحيح

**الحل**: 
1. اذهب لـ Backend في Render
2. **Environment** → عدّل `CORS_ORIGIN`
3. يجب أن يطابق URL الـ Frontend بالضبط

### المشكلة: البريد الإلكتروني لا يعمل

**السبب**: App Password غير صحيح

**الحل**:
1. تأكد من تفعيل 2FA في Gmail
2. أنشئ App Password جديد
3. حدّث `EMAIL_PASS` في Render

### المشكلة: Build Failed

**السبب**: خطأ في الكود أو المتغيرات

**الحل**:
1. افحص السجلات (Logs) في Render
2. تأكد من جميع المتغيرات البيئية موجودة
3. جرب البناء محلياً أولاً
